Project Webdevelopment

Periode:

15/09/2017 tot 30/1/2018

De opdracht/Probleemstelling:

De opdracht was om een webshop te maken. De webshop hoorde een thema te hebben, dus bijvoorbeeld een online schoenen winkel, kleding shop, game shop, ect. De producten mochten niet gehardcode worden. Wat hiermee wordt bedoel is dat we dus niet de producten zelf mochten typen, we moesten een dataset vinden en die importeren. De webshop moest voldoen aan de volgende requirements:

1) Every user (registered and unregistered) should be able to search, select, save and order products.
2) The admin should be able to create, read, update and delete information about the user accounts and products.
3) The application should validate client/server information, for example logins or e-mails.
4) There is evidence that the application is build as a result of a well thought-out design.
5) The application has been improved after testing.
6) The admin should be able to visualize statistics about the data available of the store.
7) Every registered user should be able to see the history and status of orders.
8) The registered user should be able to save a list of products for later use (the list is by default private).
9) Additional feature(s) (coordinate this beforehand with your P.O.)

Het concept:

Ons idee was om een webshop te maken die gericht zou zijn op het verkopen van video games. We hadden ons voor ogen om de webshop niet met een lokale database te maken en deze vervolgens lokaal te draaien, wij hadden ons voor ogen om de webshop online te hebben staan met een online database zodat ieder teamlid dezelfde database zou hebben tijdens het testen.

Het proces

Het proces ging helaas niet helemaal volgens plan, door verschillende redenen was het zo dat meerdere teamleden geen bijdragen konden leveren waardoor het project door twee mensen is uitgevoerd. Ook was het zo dat wij eerst een ander idee hadden voor het project, dit idee kon niet uit worden gevoerd. Door deze omstandigheden kozen wij ervoor om opnieuw te beginnen met nog 4 weken te gaan, het project duurde origineel 20 weken. Hierdoor was de werkdruk erg hoog, maar uiteindelijk is al het werk afgekomen.

Mijn bijdrage:

Ik ben in dit project verantwoordelijk geweest voor het maken van de front end wat zichtbaar is voor de eindgebruikers. Hieronder valt het weergeven van de producten, het accountmanagement gedeelte voor de gebruikers, bestel geschiedenis, wishlist, ect. De admin omgeving is door het andere teamlid gemaakt. Verder heb ik nog functionaliteit toegevoegd aan de backend. De functionaliteit die ik heb toegevoegd valt onder de requirements: 8, 7, 5, 1 (search, save), 2 (update -producten). Verder heb ik mij ook bezig gehouden met het schrijven van linq querries die nodig waren voor het efficiënt ophalen van informatie uit de database, om de snelheid van de applicatie te bevorderen.

Reflectie:

Aangezien de tijd waarin wij dit project hebben gemaakt met slechts 2 man ben ik erg tevreden over het resultaat. Ik had wel graag langer aan dit project willen werken om de website bijvoorbeeld ook voor mobiele apparaten te optimaliseren.

Technische Details:

framework: ASP.NET Core 2.0 MVC
taal: C#
platform: alle platformen die een webbrowser kunnen draaien

Mobile Game

Periode:

15/09/2017 tot 30/1/2018

De opdracht/Probleemstelling:

Dit project was niet een opdracht, ook was er geen Probleemstelling. Ik heb deze game gemaakt, omdat ik het leuk vond om een video game te maken.

Ik had voor een vorig project op school al eens een videogame moeten maken. In de tweede kwartaal van het eerste jaar moesten we een bordspel maken in pygame. Ik had toen nog geen enkele ervaring in het bouwen van een video game. Tijdens het project heb ik veel geleerd over de logica tussen verschillende objecten, hoe ze op elkaar inwerken en hoe ze met elkaar communiceren.

Deze nieuwe kennis wilde ik graag nog eens toepassen, maar dan op een heel nieuw project.

Het concept:

Ik wilde dus graag nog eens een video game maken om te kijken wat ik ervan kon maken met mijn verbeterde kennis. Mijn idee was om een sidescroller te maken die afspeelt in de jaren tachtig. Het spel moet er een beetje 8-bit achtig uitzien, alles moet er dus een beetje pixelated uitzien.

In dit spel speel je als undercover politie agent tegen een criminele bende. De bedoeling is dat je door verschillende levels heen moet gaan. Het doel is om de hoogste score te halen of de snelste tijd neer te zetten.

Het proces

Het project was ik begonnen in pygame net als het vorige project. Al snel kwam ik erachter dat pygame het niet fijn vindt als je afbeeldingen met een hoge resolutie probeert te drawen over elkaar.
Ik ben dus verder gaan kijken naar andere frameworks en ben zo bij monogame terecht gekomen. Na wat onderzoek te hebben gedaan kwam ik er al snel achter dat monogame ook mobiele applicaties ondersteunt.
Om eens wat nieuws te proberen besloot ik om verder te gaan met monogame en de game voor anroid te maken.
Ik ben eerst begonnen met bewegingen te maken met vierkanten, later ben ik hier afbeeldingen in gaan zetten voor de characters en achtergronden. Verder ben ik heel simpel te werk gegaan. Ik had van te voren een niet al te concreet plan. Tijdens het werken deed ik nieuwe ideeën op en paste ik ze toe. Ik heb in totaal elke week een uur of vijf aan het project gewerkt.

Mijn bijdrage:

Ik heb alles gemaakt, code/art/animaties.
Verder heb ik geen voorgemaakte componenten gebruikt, zoals de ingebouwde keyboard van android of een joystick. Ik vond het leuk om alles zelf uit te vinden en te maken.

Reflectie:

Over het algemeen ben ik tevreden over de game. Het ziet er mooi uit.
De enige punten die ik heb zijn:
- De game is niet helemaal af, er zitten nog een paar kleine foutjes in die ik er nooit meer uit heb gehaald vanwegen gebrek aan tijd.
- De tegenstanders zijn hardstikke dom, ik zou graag nog het gedrag van de tegenstanders willen verbeteren.

Technische Details:

framework: monogame
taal: C#
platform: android
animaties, art: photoshop
